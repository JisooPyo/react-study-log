## React.js란?

- Meta(Facebook)이 개발한 오픈소스 JavaScript 라이브러리
- 대규모 웹 서비스의 UI를 더 편하게 개발하기 위해 만들어진 기술
- Netflix, Facebook. Instagram, Notion, …

## React의 기술적인 특징

1. 컴포넌트를 기반으로 UI를 표현한다.
2. 화면 업데이트 구현이 쉽다.
3. 화면 업데이트가 빠르게 처리된다.

## 1. 컴포넌트를 기반으로 UI를 표현한다.

**컴포넌트(Component)**

- 우리말로 “구성요소”라는 뜻
- 화면을 구성하는 요소, UI를 구성하는 요소를 말함

ex. 우리가 구현해야 되는 웹 페이지가 헤더, 메인, 푸터 세 가지 부분으로 나뉘어져 있다면, 헤더 → `header.js`, 메인 → `main.js`, 푸터 → `Footer.js`라는 각각의 자바스크립트 파일에 component 라는 단위로 모듈화해서 보관할 수 있다.

웹페이지를 마치 레고 조립하듯이 여러 개의 컴포넌트들을 합쳐서 만들어낼 수가 있다.

여러 페이지에서 공통으로 헤더를 쓴다면 `header.js`를 하나만 개발하여 중복 코드를 방지할 수 있다.

## 2. 화면 업데이트 구현이 쉽다.

**업데이트란?**

- 사용자의 행동(클릭, 드래그)에 따라 웹 페이지가 스스로 모습을 바꿔 상호작용 하는 것

<aside>

리액트는 "선언형 프로그래밍"이라는 방식으로 동작하기 때문에 화면 업데이트 구현이 쉽다.

업데이트를 위한 복잡한 동작을 직접 정의할 필요 없이 특정 변수의 값을 바꾸는 것만으로도 화면을 업데이트 시킬 수 있다.

</aside>

<aside>
선언형 프로그래밍이란?

불필요한 과정은 생략하고 내가 이루고자 하는 목적만 간결히 명시하는 방법

↔️ 명령형 프로그래밍이란?

선언형 프로그래밍의 반대되는 개념. 목적을 이루기 위한 일련의 모든 과정을 하나하나 다 설명하는 방식

</aside>

## 3. 화면 업데이트가 빠르게 처리된다.

**브라우저의 렌더링 과정(Critical Rendering Path)**

![image.png](attachment:a5828f37-3068-495a-b973-442f49feb1da:4c95d482-05be-491d-9834-87284876a2ed.png)

- **DOM(Document Object Model)**: HTML 문서를 일종의 객체 모델로 변환한 것.
- **Render Tree**: 웹 페이지의 청사진
- **CSSOM**: 요소들의 스타일과 관련된 모든 정보
- **Layout**: 요소의 배치를 잡는 작업
- **Painting**: 실제로 화면에 그려내는 과정

화면의 업데이트는 JavaScript가 DOM을 수정할 때 발생한다. DOM이 수정되면 브라우저는 Critical Rendering Path의 전체 단계를 다시 수행한다.

- 렌더 트리(Render Tree) 재생성 → 레이아웃(Layout) 재계산 → 페인팅(Painting) 재수행

<aside>
Reflow

- DOM 수정으로 인해 레이아웃이 다시 계산되는 과정

Repaint

- DOM 수정으로 인해 페인팅 과정이 다시 수행되는 것

레이아웃과 페인팅 과정은 비교적 오래 걸리는 작업이다.

</aside>

DOM을 짧은 시간 동안 수천 번 수정하면 렌더 트리 이후 과정이 수천 번 반복되어 성능이 크게 악화된다. 심한 경우 브라우저가 응답 없음 페이지를 표시하기도 한다. 이를 개선하려면 다양한 업데이트를 동시에 모아 한 번에 수정하면 된다. 하지만 이는 서비스의 규모가 커질 수록 점점 힘들어진다.

⇒ React는 다양한 업데이트를 동시에 모아 한 번에 수정하는 식으로 자동으로 처리해준다.

React는 내부적으로 가상 DOM을 이용하고 있다.

<aside>
Virtual DOM이란?

- DOM을 자바스크립트 객체로 흉내낸 것.
- React는 업데이트가 발생하면 실제 DOM을 수정하기 전에 이 가상의 복제판 DOM에 먼저 반영해본다.
- 가상의 DOM에 동시에 발생한 업데이트들이 다 모이면 한 번에 실제 DOM에 반영한다.
</aside>
